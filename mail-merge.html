Lecture 02

메일 머지는 한 장의 표로
여러 통의 메일을 만듭니다

템플릿 메일의 문장은 그대로 두고, Google 스프레드시트의 각 행에 있는 이름, 학번, 성적 같은 값만 정해진 자리에 끼워 넣어 개인별 메일을 발송합니다.

  • 1HTML 템플릿은 모든 학생에게 공통으로 보낼 메일 형식을 담습니다.
  • 2Google 스프레드시트는 학생별로 달라지는 데이터를 한 행씩 담습니다.
  • 3Apps Script는 행을 읽고 템플릿의 빈칸을 채운 뒤 메일을 보냅니다.

Core Idea

메일 머지는 템플릿, 표, 스크립트가 만나는 작업입니다

한 사람에게 쓰는 메일을 여러 번 복사하는 대신, 공통 문장과 개인 데이터를 분리해 자동으로 합칩니다.

HTML 템플릿

“{{name}} 학생”, “{{score}}점”처럼 바뀔 자리를 표시해 둡니다. 문장 구조와 디자인은 한 번만 작성합니다.

Sheet 학생 데이터

각 행은 한 명의 학생입니다. 이름, 학번, 성적, 이메일 주소처럼 개인별 값이 열로 정리됩니다.

Script 자동 발송

Apps Script가 행을 하나씩 읽고 템플릿의 표시 자리에 값을 넣은 뒤 Gmail로 개별 메일을 보냅니다.

Code Pair

Apps Script 프로젝트에는 GAS와 HTML 파일만 있으면 됩니다

`Code.gs`는 Google 서비스와 자동화 흐름을 맡고, `mail.html`은 메일 본문의 모양을 맡습니다. 메일 머지는 이 두 파일이 학생 데이터와 만나는 구조입니다.

GAS Code.gs

  • 스프레드시트에서 학생 행을 읽습니다.
  • 한 행을 이름, 학번, 성적, 이메일 데이터로 정리합니다.
  • HTML 템플릿에 데이터를 넘기고 Gmail 발송을 실행합니다.
Code.gs
├─ read spreadsheet rows
├─ make student data
├─ fill mail.html
└─ send with Gmail

HTML mail.html

  • 모든 학생에게 공통으로 보낼 메일 문장을 담습니다.
  • 이름, 학번, 성적처럼 바뀔 자리만 표시해 둡니다.
  • Apps Script가 전달한 값으로 개인별 본문이 완성됩니다.
mail.html
├─ subject template
├─ {{name}}
├─ {{studentId}}
└─ {{score}}, {{grade}}
Google Sheet

Wrap-up

메일 머지의 핵심은 반복 문장을 데이터와 분리하는 것입니다

템플릿은 문장의 형태를 맡고, 스프레드시트는 개인별 값을 맡고, Apps Script는 둘을 합쳐 발송합니다. 이 구조를 이해하면 출석 안내, 성적 안내, 수료증 발송 같은 반복 업무를 자동화할 수 있습니다.
