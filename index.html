Lecture 01

웹 앱은 세 가지 언어로 생각한다

바이브 코딩을 시작하기 전, 브라우저가 무엇을 받아 화면을 만드는지 먼저 잡습니다. 가장 작은 웹 앱은 HTML, CSS, JavaScript라는 세 역할의 합으로 움직입니다.

  • 1HTML은 화면에 무엇이 있는지 정하는 구조입니다.
  • 2CSS는 그 구조가 어떤 모습과 분위기를 갖는지 정합니다.
  • 3JavaScript는 클릭, 입력, 변화처럼 살아 움직이는 행동을 만듭니다.

Core Idea

브라우저 앱은 보통 3개의 코드 파일로 나뉩니다

실제 프로젝트에서는 역할을 분리하면 읽기 쉽고 고치기 쉽습니다. 그래서 처음에는 세 파일이 따로 있는 그림으로 이해하는 것이 좋습니다.

HTML 구조

제목, 문단, 버튼, 이미지, 입력창처럼 화면에 놓일 재료를 배치합니다. 브라우저는 HTML을 읽고 문서의 뼈대를 먼저 만듭니다. 

<main class="app">
  <h1>오늘의 웹 앱</h1>
  <p id="message">버튼을 눌러 보세요.</p>
  <button id="run">시작</button>
  <img src="cover.png" alt="">
</main>

CSS 스타일

색, 간격, 글꼴, 배치, 반응형 레이아웃을 정합니다. 같은 HTML이라도 CSS가 달라지면 완전히 다른 경험이 됩니다. 

body {
  margin: 0;
  font-family: sans-serif;
  background: #f7f5f0;
  color: #2b2b2b;
}

JS 동작

사용자의 행동에 반응하고 데이터를 바꾸며 화면을 갱신합니다. 버튼을 눌렀을 때 문구가 바뀌는 작은 변화도 JavaScript가 맡습니다. 

const button = document.querySelector("#run");
const message = document.querySelector("#message");
button.addEventListener("click", () => {
  message.textContent = "화면이 바뀌었습니다.";
  button.textContent = "완료";
});

Single HTML

CSS와 JS는 HTML 안에 포함될 수 있습니다

`style.css`의 내용은 HTML의 `<style>` 태그 안으로, `app.js`의 내용은 `<script>` 태그 안으로 들어갈 수 있습니다. 그러면 파일은 하나지만 세 역할은 그대로 남습니다.

HTML index.html

문서 구조를 담고 CSS와 JS가 들어올 자리를 갖습니다.

CSS style.css

색, 간격, 배치 같은 스타일 규칙입니다.

JS app.js

클릭과 상태 변화를 처리하는 동작 코드입니다.

HTML single-file.html
1 file
HTML 구조
CSS 스타일
JS 동작
분리 방식

`index.html`, `style.css`, `app.js`처럼 파일을 나눕니다. 규모가 커질수록 유지보수에 유리합니다.

포함 방식

HTML 안의 `<style>`과 `<script>`에 CSS, JS를 넣습니다. 작은 예제와 배포용 데모에 유리합니다.

핵심 감각

HTML은 뼈대, CSS는 스타일, JS는 행동입니다. 하나로 합쳐도 이 세 역할은 사라지지 않습니다.

Wrap-up

바이브 코딩은 “요청” 전에 “구조”를 보는 힘에서 시작합니다

AI가 코드를 만들어 주더라도, 우리가 읽어야 할 기본 단위는 그대로입니다. 화면의 재료는 HTML, 분위기는 CSS, 반응은 JavaScript입니다. 이 세 역할을 구분할 수 있으면 단일 HTML 데모도, 파일이 나뉜 프로젝트도 훨씬 편하게 다룰 수 있습니다.
