Lecture 06

GitHub Pages로 웹 앱을 공개합니다

HTML, CSS, JavaScript로 만든 정적 웹 앱은 서버를 직접 준비하지 않아도 GitHub 저장소에서 바로 공개할 수 있습니다.

배포 흐름

저장소에서 공개 주소까지 네 단계입니다

파일을 올리고, Pages 설정을 켜고, 배포 소스를 고르면 GitHub가 공개 주소를 만들어 줍니다.

1

파일 올리기

index.html이 저장소의 시작점이 됩니다.

2

Pages 열기

저장소의 Settings에서 Pages 메뉴로 들어갑니다.

3

소스 고르기

Deploy from a branch, main, /(root)를 선택합니다.

4

주소 확인

배포가 끝나면 Visit site로 공개 페이지를 엽니다.

실습 체크

작은 정적 웹 앱이라면 이것만 확인합니다

GitHub Pages는 정적 파일을 공개하는 도구입니다. 비밀키와 서버 코드는 올리지 않는 감각이 중요합니다.

✓첫 화면 파일은 index.html로 둡니다.
✓이미지와 CSS 경로는 상대 경로로 연결합니다.
✓배포 소스는 main 브랜치의 /(root)부터 시작합니다.
✓수정 후 push하면 Pages가 다시 배포합니다.
핵심 감각

배포는 코드를 새로 쓰는 일이 아니라, 브라우저가 읽을 수 있는 파일을 공개 위치에 올려 두는 일입니다.
